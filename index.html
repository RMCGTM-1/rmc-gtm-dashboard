🚨 Call tracking must be live before ads go live. Without it, Google Ads has no conversion signal and Smart Bidding optimizes toward clicks, not calls. 90-second minimum duration is your quality filter — this is the single most important configuration decision before launch.
Setup Steps — In Order
1. Create CallRail account + number pool
~30 min
Pool of 4–6 tracking numbers. DNI swaps displayed number by traffic source.
2. Install DNI snippet in Lovable index.html
~15 min
Must go in <head> of index.html root — not just a component. React SPA needs it at root level.
3. Connect CallRail → Google Ads
~20 min
CallRail Settings → Integrations → Google Ads. Passes call conversions back for Smart Bidding.
4. Set 90-second minimum duration threshold
Decision ✓
Import only calls ≥ 90 seconds as conversions in Google Ads. Filters hangups, wrong numbers, misdials. Revisit at Day 30 based on actual duration distribution.
5. Enable call recording + disposition tagging
Phase 2
Tag calls: Quote Presented / Sold / No Answer / Wrong Number / Hangup. Enables value-based bidding later.
Platform + Consent Notes
CallRail
Recommended
~$45–65/mo · Native Google Ads integration · DNI · Call recording · Keyword-level attribution · Right-sized for <$5K/mo budget.
MA + TX Consent
Both states are one-party consent for call recording. Best practice: add brief disclosure ("This call may be recorded for quality purposes") to cover edge cases if calls cross state lines.
Conversion Actions in Google Ads
Primary: Website call via CallRail ≥ 90 sec · drives Smart Bidding
Secondary: Phone call from ad (Google native) · catches SERP call extension clicks
